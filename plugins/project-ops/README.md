# project-ops

Claude Code plugin for GitHub Project V2 management — auto-discovery, opinionated board setup, and issue hygiene hooks.

## Features

- **Auto-discovery**: Detects org/repo from git remote, discovers project field IDs via GraphQL
- **Project bootstrap**: Creates new GitHub Project V2 boards with opinionated fields, views, and linked repos
- **Issue hygiene hooks**: Blocks checkbox task lists (enforces native sub-issues), reminds about project fields and labels
- **Portable scripts**: 11 shell scripts for blocking relationships, sub-issues, status queries, and more
- **Skills & commands**: Both Claude-invocable skills and user-invocable slash commands

## Installation

```bash
claude plugin add https://github.com/WonderForgeLabs/project-ops
```

## Quick Start

1. **Configure** the plugin for your repo:
   ```
   /project-ops:configure
   ```
   This detects your org/repo, lists available projects, and writes a config file.

2. **Or bootstrap** a new project board:
   ```
   /project-ops:bootstrap
   ```
   Creates a GitHub Project V2 with opinionated fields (Priority, Critical Path, Sprint, dates), views (Roadmap, Sprint Board, etc.), and links your repo.

3. **Check status**:
   ```
   /project-ops:status
   ```

4. **Find work**:
   ```
   /project-ops:find-work
   ```

5. **Close an issue and update the board**:
   ```
   /project-ops:update-roadmap 123
   ```

## Components

### Commands

| Command | Description |
|---------|-------------|
| `/project-ops:configure` | Interactive setup — detect org/repo, select project, write config |
| `/project-ops:bootstrap` | Create a new GitHub Project V2 with opinionated defaults |
| `/project-ops:status` | Show board overview grouped by critical path + unblocked work |
| `/project-ops:find-work` | Find the next highest-priority unblocked issue |
| `/project-ops:update-roadmap <N>` | Close issue, check what got unblocked, report results |

### Skills (auto-invoked by Claude)

| Skill | Triggers on |
|-------|-------------|
| `project-status` | "what's the status", "where are we", checking roadmap |
| `find-work` | "what should I build next", looking for work |
| `update-roadmap` | completing work, closing issues, merging PRs |

### Hooks

| Event | Hook | Purpose |
|-------|------|---------|
| PreToolUse | Block checkbox task lists | Enforce native GitHub sub-issues |
| PreToolUse | Code review reminder | Remind before PR creation |
| PostToolUse | Issue field reminder | Remind to set relationships, priority, labels |

### Scripts

| Script | Purpose |
|--------|---------|
| `project-config.sh` | Shared config loader (auto-detect or read `.claude/project-ops.yaml`) |
| `bootstrap-project.sh` | Create project with fields, views, linked repo |
| `add-blocker.sh` | Add blocking relationship between issues |
| `remove-blocker.sh` | Remove blocking relationship |
| `add-sub-issue.sh` | Add sub-issue relationship |
| `show-blockers.sh` | Show blocking/blocked-by/sub-issues for an issue |
| `find-unblocked-work.sh` | List open issues with no open blockers |
| `project-status.sh` | Show items grouped by critical path |
| `items-by-status.sh` | Show items grouped by status column |
| `milestone-progress.sh` | Show milestone open/closed counts |
| `project-views.sh` | Show view configurations |

## Configuration

The plugin stores per-project config in `.claude/project-ops.yaml`. Commit this file to share the configuration with your team. The file uses YAML frontmatter:

```markdown
---
org: WonderForgeLabs
repo: forge
project_number: 2
---

# Project Ops Configuration

This file was generated by `/project-ops:configure`.
```

If no config file exists, the plugin auto-detects org/repo from the git remote.

## Prerequisites

- [GitHub CLI](https://cli.github.com/) (`gh`) authenticated with appropriate scopes
- Git repository with a GitHub remote
- Organization-level GitHub Project V2 (or use `/project-ops:bootstrap` to create one)

## Default Project Structure

When using `/project-ops:bootstrap`, the following opinionated structure is created:

**Fields:**
- Priority (P0-P4 single select)
- Critical Path (single select, empty — add groups later)
- Sprint (iteration)
- Start Date, Target Date (date)
- Design Doc (text)

**Views:**
- Roadmap (roadmap layout, grouped by Critical Path)
- What can I work on? (board, sorted by Priority)
- Critical Paths (table, grouped by Critical Path)
- Sprint Board (board, grouped by Sprint)
- Dependency Order (table, sorted by Priority)
